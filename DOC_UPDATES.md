### Key Problems

1. **No modern docs in git** — The only committed docs are legacy HTML with deeply outdated content
2. **Dead references everywhere** — SourceForge, CVS, `install/` directory, PHP 7.4, Internet Explorer, Cygwin
3. **Wrong version numbers** — SysAdmin guide says v1.9.9, others say v1.9.13 or nothing
4. **No CHANGELOG** — No structured release history anywhere
5. **UPGRADING.html** is HTML and describes the pre-v1.9.13 installer
6. **SECURITY.md** is 2 lines — doesn't follow GitHub security policy best practices
7. **README.md** links to legacy HTML docs via `htmlpreview.github.io` hack
8. **No GitHub Pages or docs site** — docs only readable by browsing HTML files on GitHub
9. **Abandoned Perl doc toolchain** — `php2html.pl`/`sql2html.pl` generate files nobody maintains
10. **No CONTRIBUTING.md** in root (GitHub looks for it there)
11. **Internal planning docs clutter the root** — PRD.md, ROADMAP.md, etc. confuse end users
